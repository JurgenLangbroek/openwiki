# Gateway passthrough reads are policed per downstream tool

Glean's MCP gateway exposes underlying datasources (Jira, Confluence, Gmail, Calendar) only through a `run_tool` passthrough, which is write-shaped by name and description and therefore hard-blocked by the read-only tool policy (ADR-0001). To give Exploration full read reach into those datasources without weakening the read-only-observer invariant, openwiki never exposes raw `run_tool` to the agent; instead the connector offers a Gateway Datasource Read tool that resolves the *downstream* tool from the `find_skills` catalog and applies the standard deny-by-default read-only policy to that downstream tool's name, description, and annotations. Read-only downstream tools pass; write-shaped or unclassifiable ones are never callable, and `allowedTools` cannot override a write-shaped downstream denial.

**Considered options:** allowlisting `run_tool` wholesale was rejected because its only marginal capability over downstream policing is unattended writes to work systems — exactly the incident class ADR-0001 exists to prevent.
